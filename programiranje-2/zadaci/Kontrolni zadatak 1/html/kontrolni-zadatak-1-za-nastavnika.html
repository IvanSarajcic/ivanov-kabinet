I. Питања са кратким одговором
1. Дефиниши алгоритам.
Одговор: Алгоритам је прецизан опис поступка за решавање неког проблема у коначном броју корака.
(Прихватити и сличне дефиниције које помињу коначност, прецизност и решавање проблема).
2. Шта значи да је алгоритам парцијално коректан?
Одговор: Значи да ако се алгоритам заустави, резултат који врати је тачан.
Не гарантује да ће се зауставити.
3. Пример за који функција Min НЕ ради исправно:
int Min(int a, int b, int c) { if (a < b) return a; else if (b < c) return b; else return c; }
Одговор: (2, 5, 1) или било који пример где је 'a' мање од 'b', али није најмањи.
Објашњење: 2 < 5 је тачно, па функција враћа 2, иако је 1 најмањи.
4. Префиксни збирови за {5, 3, 2, 1}
Одговор: b = { 5, 8, 10, 11 }
5. Шта је "грешка за 1" (off-by-one error)?
Одговор: Грешка када се петља изврши 1 пут превише/премало или се приступи индексу ван граница за 1.
II. Питања вишеструког избора
6. Који од наведених исказа најбоље описује тоталну коректност алгоритма?
A. Алгоритам је парцијално коректан и зауставља се за све исправне улазе.
7. Шта је проблем са овим кодом ако је циљ да се попуни цео низ b?
B. Петља се не извршава довољан број пута (последњи елемент неће бити израчунат).
Петља иде до n-1, дакле последњи упис је на позицију n-1, али прескаче се рачунање за неке елементе или се не попуни све како треба (у зависности од интерпретације, али B је најтачнији опис логичке грешке у контексту задатка).
8. Шта ће се догодити са следећим кодом ако је n велики позитиван број?
B. Може доћи до прекорачења (overflow) ако збир постане превелик за тип int.
9. Зашто није довољно тестирати само случај А (сортиран низ)?
A. Зато што је то већ сортиран низ, па не проверава логику замене места.
10. Која врста верификације подразумева математички доказ исправности без покретања кода?
B. Статичка верификација.
