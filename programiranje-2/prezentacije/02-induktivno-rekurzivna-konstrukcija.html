Честе грешке у програмима

Преглед најчешћих извора грешака

Проблем комуникације

  • Задатак није довољно прецизно постављен
  • Различито разумевање задатка
  • Недовољно пажљиво читање спецификације
  • Погрешно разумевање задатка

Грешка погрешног избора типа

  • Бројевни тип не може да представи све вредности
  • Долази до прекорачења (overflow)
  • Посебно опасно: тип је добар за улаз/излаз, али не за међурезултате

Грешка за 1 (Off by one)

Питање: Стубови се постављају на свака 2 метра. Ограда је дуга 20 метара. Колико стубова?

Одговор: 11, не 10!

Програмерски: Обрадити елементе од индекса a до b. Колико елемената? b-a+1

Buffer Overflow

  • Приступ елементима ван граница низа
  • Низ од n елемената: позиције 0, 1, ..., n-1
  • Критична позиција: n (не постоји!)

Математичка индукција

Кључна идеја у конструкцији алгоритама

Принцип индукције

За својство P природних бројева:

(P(0) ∧ (∀n)(P(n) ⇒ P(n+1))) ⇒ (∀n)(P(n))

  • База: P(0) је тачно
  • Корак: P(n) ⇒ P(n+1)
  • Закључак: P важи за све природне бројеве

Интуитивно објашњење

  • База: 0 има својство P
  • Корак: ако n има P, онда и n+1 има P
  • Дакле: 0 → 1 → 2 → 3 → ...
  • Можемо стићи до било ког природног броја

Индуктивни приступ

Основни метод конструкције алгоритама

Идеја приступа

  • Решење проблема веће димензије
  • Користи решење проблема мање димензије
  • За елементарне проблеме: директно решење
  • Ако се димензија увек смањује → заустављање

Два начина имплементације

Итеративно

  • Петље
  • Ажурирање променљивих
  • Индуктивно

Рекурзивно

  • Функција позива саму себе
  • Мањи проблем
  • Рекурзивно

Пример: Збир низа

Класичан пример индуктивне конструкције

Индуктивна идеја

  • Знамо збир празне серије: 0
  • Знамо збир од k елемената
  • Како израчунати збир од k+1 елемената?
  • Додати нови елемент на постојећи збир!

Имплементација


int zbir = 0;
for (int i = 0; i < a.Length; i++)
    zbir = zbir + a[i];
  • База: zbir = 0 (празна серија)
  • Корак: zbir += a[i] (додај нови елемент)

Варијанта


int zbir = a[0];  // база: први елемент
for (int i = 1; i < a.Length; i++)
    zbir = zbir + a[i];

База може бити и једночлани низ!

Веза са доказивањем коректности

  • Индуктивна конструкција ↔ Доказивање коректности
  • Формални оквир: Хорова логика
  • Ми ћемо користити неформалне доказе
  • Следећа тема: Инваријанте петље

Поједностављења за учење

Игнорисаћемо ограничења рачунара:

  • Неограничен опсег бројева
  • Максимална прецизност реалних бројева
  • Без overflow/underflow грешака

Фокус на логику, не на техничке детаље

Кључне поуке

  • Исправан алгоритам ≠ исправан програм
  • Познати грешке и како их избећи
  • Математичка индукција је основа
  • Два приступа: итеративни и рекурзивни
  • Индуктивна конструукција води ка доказивању коректности
